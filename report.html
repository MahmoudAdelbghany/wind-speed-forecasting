Wind Speed Time-Series Forecasting: A Comparative Study of
Machine Learning and Deep Learning Approaches

Research Internship Report — June 2026

1. Problem Statement

Accurate wind speed forecasting is critical for wind energy integration, grid stability, and meteorological planning. This study addresses short-term univariate wind speed prediction using 5,000 sequential measurements (range: 0–18.7 m/s). We benchmark two machine learning models (Echo State Network, Gaussian Process Regression) and two deep learning models (LSTM, Transformer), then systematically improve performance through architecture tuning, extended input windows, and ensemble methods.

4. Methodology & Implementation

4.1 Data Pipeline

Data was normalized to [0,1] via MinMaxScaler. Supervised windows of size k were constructed: each input of k consecutive values predicts the next. An 80/20 sequential split preserved temporal order. For GPR, training was limited to the most recent 1,000 points to manage O(N³) complexity.

4.2 Training Configuration

All DL models used Adam optimizer, ReduceLROnPlateau scheduling, gradient clipping (max_norm=1.0), and early stopping (patience=7). ESN used sparse reservoir connectivity (10% sparsity) with hyperparameter search over reservoir size (100–250), spectral radius (0.9–0.99), and leakage (0.15–0.5).

Best individual model: Echo State Network (window=21), RMSE = 0.2984.

The ESN's superiority is consistent with EDA findings — its reservoir dynamics are inherently designed to capture the strong serial autocorrelation (r = 0.974) present in this dataset, while requiring minimal training computation compared to deep learning alternatives.

6. Limitations

LimitationImpact
Small dataset (5,000 samples)DL models (LSTM, Transformer) cannot fully leverage their capacity; risk of overfitting
Univariate forecasting onlyExcludes wind direction, temperature, pressure — variables that could improve accuracy
Single train/test splitSingle-split RMSE (0.30) may be optimistic; walk-forward validation yields RMSE ≈ 1.0, reflecting more realistic production conditions
No seasonal decompositionDataset lacks timestamps; cannot model diurnal or seasonal patterns
GPR scalabilityO(N³) complexity limits GPR to 1,000 training points; larger windows degrade performance

7. Conclusion

For univariate wind speed forecasting with strong autocorrelation (r = 0.974), Echo State Networks outperform both traditional ML and deep learning architectures. Extending the input window from 7 to 21 steps — motivated by EDA autocorrelation analysis — yields a 5.6% RMSE improvement. The best individual model achieves RMSE = 0.2984. Limitations include the small dataset size, univariate-only input, and the gap between single-split and walk-forward evaluation metrics.
