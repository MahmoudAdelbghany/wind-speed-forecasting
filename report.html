Wind Speed Time-Series Forecasting: A Comparative Study of
Machine Learning and Deep Learning Approaches

Research Internship Report — June 2026

1. Problem Statement

Predicting wind speed matters a lot for wind energy — grid operators need to know how much power to expect, and weather services need accurate short-term forecasts. The goal here is simple: given the last k wind speed readings, predict the next one. We work with a single-turbine dataset of 5,000 measurements (ranging from calm 0 m/s to strong 18.7 m/s gusts) and compare four models: two from classical machine learning (Echo State Network, Gaussian Process Regression) and two from deep learning (LSTM, Transformer).

4. Methodology & Implementation

4.1 Data Pipeline

The raw data was scaled to [0, 1] using MinMaxScaler — this keeps the neural network training stable. We then slid a window of size k across the series: each window of k values becomes one input sample, and the value right after the window is the target. The first 80% of samples go to training, the rest to testing, in order (no shuffling — that would leak future information into the past). For GPR, we limited training to the last 1,000 points because the kernel matrix grows cubically with dataset size.

4.2 Training Details

Deep learning models used Adam optimizer with a learning rate scheduler that cuts the rate in half when validation loss plateaus. We also used gradient clipping (to prevent exploding gradients) and early stopping (training halts if validation loss doesn't improve for 7 epochs). The ESN was built from scratch: a sparse random recurrent network with 150 units, where only the output weights are trained via Ridge regression. We searched over a small grid of hyperparameters (reservoir size, spectral radius, leakage rate) and picked the best combo.

4.3 What we changed and why

After seeing the baseline results, two things stood out: (1) the deep learning models were underperforming, and (2) the EDA showed strong autocorrelation even at lag 21. So we made two changes:

ChangeWhat we didWhy
TuningAdded dropout, early stopping, LR scheduling, ESN hyperparameter searchModels were overfitting or using poor defaults
Bigger windowWent from 7 to 21 stepsACF at lag 21 is still 0.92 — the models were missing useful history

5.4 Bottom line

Best individual model: ESN with window=21 (RMSE = 0.2984). It wins because the reservoir architecture is a natural fit for this data — it captures the strong serial correlation without the training overhead of full backpropagation. The GPR is a close second (0.3018), and the Transformer came third after tuning (0.3583). The LSTM never got competitive on this dataset.

6. Limitations

IssueWhy it matters
Small dataset (5,000 points)Deep learning models like LSTM and Transformer need more data to learn properly. With 5,000 samples, they're essentially guessing.
Only wind speed — no other variablesReal wind prediction uses temperature, pressure, wind direction, humidity. We only had speed, which limits accuracy.
Single train/test splitOne split can be lucky or unlucky. When we tested with rolling cross-validation, errors were about 3× higher (RMSE ≈ 1.0 vs 0.30), suggesting the single-split number is optimistic.
No timestamps in the dataWind has daily and seasonal patterns (stronger at night, different in winter vs summer). Without timestamps, we can't model any of that.
GPR doesn't scaleThe kernel matrix is O(N³), so we had to cap training at 1,000 points. Bigger windows made this worse — GPR couldn't be used for windows above 14.

7. Conclusion

The short version: for this particular wind speed dataset, an Echo State Network with a 21-step input window works best (RMSE = 0.2984). It beats the GPR (0.3018), the Transformer (0.3583), and the LSTM (0.4965). The ESN wins because the data has very strong autocorrelation (r = 0.974), and ESNs are specifically designed to exploit exactly that kind of temporal structure — without the training cost of deep learning.

The bigger picture: extending the window from 7 to 21 steps gave a 5.6% improvement, which came directly from looking at the autocorrelation function in the EDA. That's the kind of insight that matters more than picking a fancier model. The main limitations are the small dataset, the lack of additional meteorological variables, and the gap between single-split and walk-forward evaluation numbers.
