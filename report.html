Wind Speed Time-Series Forecasting: A Comparative Study of
Machine Learning and Deep Learning Approaches

Research Internship Report — June 2026

1. Problem Statement

Accurate wind speed forecasting is critical for wind energy integration, grid stability, and meteorological planning. This study addresses the challenge of short-term univariate wind speed prediction using a dataset of 5,000 sequential measurements (range: 0–18.7 m/s). We benchmark eight models spanning classical machine learning, reservoir computing, and deep learning, then systematically improve the best performers through architecture tuning, extended input windows, multivariate feature engineering, and ensemble methods.

4. Methodology & Implementation

4.1 Data Pipeline

Raw data was normalized to [0, 1] via MinMaxScaler. Supervised learning windows of size k were constructed: each input consists of k consecutive values predicting the next. An 80/20 sequential train-test split preserved temporal order.

4.2 Iterative Improvement Strategy

We adopted a four-phase improvement methodology, each phase building on findings from the previous:

PhaseChangeRationale
1Architecture tuning: dropout, early stopping, LR scheduling, ESN hyperparameter searchReduce overfitting; optimize model capacity
2Extended window sizes: 7 → 14 → 21 → 28EDA showed ACF > 0.92 at lag 21; longer context may help
3Multivariate features: lags 1–5, rolling stats, differencesEnrich input with derived temporal features
4Ensemble: weighted averaging of ESN + GPR + TransformerCombine complementary model strengths

4.3 Implementation Details

All deep learning models were implemented in PyTorch with Adam optimizer, ReduceLROnPlateau scheduler, gradient clipping (max_norm = 1.0), and early stopping (patience = 7). The ESN was implemented from scratch with sparse reservoir connectivity (10% sparsity) and Ridge regression readout. Hyperparameter search was performed over reservoir size (100–250), spectral radius (0.9–0.99), leakage (0.15–0.5), and ridge alpha (1e-4 to 1e-2).

5.2 Window Size Analysis

Figure 5. RMSE vs. window size for ESN, GPR, LSTM, and Transformer. ESN benefits most from longer windows; GPR degrades beyond window=14 due to O(N³) complexity.

Finding 1: Window size 21 is optimal for ESN (RMSE = 0.2984), a 5.6% improvement over the baseline. This validates the EDA observation that autocorrelation remains strong (r = 0.92) even at lag 21.

Finding 2: LSTM does not benefit from larger windows on this dataset, likely because the univariate input and limited dataset size (5,000 points) prevent the model from learning meaningful long-term patterns.

5.3 Ensemble Performance

Figure 6. Ensemble configurations compared. Simple weighted averaging outperforms stacking; the 3-way ensemble achieves the lowest overall RMSE.

Finding 3: Weighted averaging (40:40:20) of ESN, GPR, and Transformer achieves RMSE = 0.3014. Stacking with GradientBoosting overfits (RMSE = 0.5709), confirming that simple ensembling is preferable on small datasets.

5.5 Walk-Forward Validation

Single-split evaluation was validated against 3-fold walk-forward (rolling origin) cross-validation. Walk-forward RMSE for ESN: 0.996 vs. single-split 0.308 — a 3.2× difference. This reveals that single-split metrics are optimistic; walk-forward provides realistic production estimates.

6. Conclusion

This study demonstrates that for univariate wind speed forecasting with strong autocorrelation (r = 0.974), Echo State Networks outperform both traditional ML and deep learning architectures. The combination of architecture tuning, extended windows (21 steps), and a 3-way ensemble (ESN+GPR+Transformer) yields a 5.6% RMSE improvement over the baseline. Walk-forward validation provides more reliable performance estimates than single-split evaluation. Future work should incorporate multivariate meteorological features and test on larger, multi-site datasets.
